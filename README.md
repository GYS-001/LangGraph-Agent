# LangGraph-Agent
## 基于 LangGraph 的专属 AI Agent 设计与实现
为“YOLOv8+SAM2 智能视频监控”项目设计的专属 AI Agent，实现从“手动敲命令”到“一句话运行项目”的效率飞跃。
##  项目背景

在深度学习项目开发中，每次运行都要重复“激活环境 → 进入目录 → 运行脚本 → 查看日志”的繁琐流程。为解决这个问题，决定为项目搭建一个专属 AI Agent，目标是通过简单的自然语言指令来驱动整个流程。

##  核心技术方案

本项目使用 LangGraph 框架，基于 **模型层 (Brain)、工具层 (Hands)、编排层 (Orchestration)** 的设计思想构建 Agent。

**核心工作流**：用户输入指令 `运行项目` → Agent 接收并分析意图，决定调用工具 → 工具执行，使用特定 Python 环境运行目标脚本，并实时捕获输出 → 工具返回执行结果，Agent 进行总结并反馈给用户

```
用户指令
│
▼
┌─────────────────────────┐
│ LangGraph 工作流 │
│ (Agent思考 → 决定行动) │
└─────────────┬───────────┘
│ 调用
▼
┌──────────────────────────┐
│ 本地大模型 (Ollama) │
│ llama3.1:8b-instruct-q4 │
└─────────────┬────────────┘
│ 执行
▼
┌──────────────────────────┐
│ 自定义工具 (Python函数) │
│ · run_project │
│ · run_project_with_ │
│ monitor │
└──────────────────────────┘
```

##  环境与工具

| 类别 | 技术 | 说明 |
|------|------|------|
| 框架 | LangGraph | 构建有状态、多步骤的 Agent 工作流 |
| 模型 | Llama 3.1 8B (via Ollama) | 负责理解指令并决策，工具调用稳定 |
| 环境隔离 | Anaconda (Conda) | Agent 环境和项目环境相互隔离，互不干扰 |
| 系统调用 | Python `subprocess` | Agent 的“手”，允许它在系统层面执行命令 |

## 关键设计决策

*   **模型选型**：**弃用 `qwen2.5-coder:7b`，改用 `llama3.1:8b-instruct-q4`**。前者虽擅长代码，但在工具调用的稳定性上不够理想；后者工具调用极度稳定，是 Agent 决策的可靠基石。这体现了实践中“适配优于参数”的原则。
*   **环境隔离**：Agent 本身在一个 Conda 环境 (`local_agent`)，但它能通过绝对路径调用另一个 Conda 环境中的 Python，去执行项目脚本。这保证了二者环境的绝对独立，不会相互污染。
*   **“硬直连”兜底**：在调试过程中，为规避模型偶尔不按预期调用工具的问题，设计了一个巧妙的“硬直连”环节，直接对用户指令中的关键词（如“运行项目”）进行拦截，确保核心功能绝对可靠。
